# DocAtHome Patient - Mobile Client

A modern Android application for patients to manage their at-home medical appointments, view medical reports, and maintain their healthcare profile. This mobile app is the client-side interface for the CareHouse Healthcare Management System Patient backend.

**Note:** This implementation covers the core patient features, not all backend functionality. It serves as a learning resource demonstrating clean architecture principles and how to integrate a mobile app with microservices-based backend systems.

## Overview

DocAtHome Patient is a mobile app that enables patients to:
- Register and authenticate securely via backend JWT authentication
- Schedule and manage medical appointments
- Access medical reports and records stored in the backend
- Maintain and update their health profile
- Connect with healthcare providers offering at-home services
- Upload and manage medical documents

**Backend Integration:** This app connects to the [CareHouse Backend](https://github.com/Mohamed-Nafae/Backend-Healthcare-Management-System-for-CareHouse-Project/README.md) Patient subsystem (Ports 5000-5005)

## Data Models

These models are synchronized with the backend MongoDB collections:

### Patient (from Backend - Port 5003)
```kotlin
data class Patient(
    val appointments: List<String>?,          // Appointment IDs
    val birth_date: LocalDate,
    val email_address: String,                // Unique identifier
    val first_name: String,
    val gender: String,
    val imageBitmap: Bitmap?,                 // Profile picture (from Port 5004)
    val last_name: String,
    val location: Location,
    val medical_folder: String,               // Reference to medical folder (Port 5002)
    val medical_reports: List<String>?,       // Report IDs (from Port 5005)
    val password: String?,                    // Hashed in backend
    val phone_number: String                  // Unique identifier
)
```

### Appointment (from Backend - Port 5001)
```kotlin
data class Appointment(
    val _id: String,                          // MongoDB ObjectId
    val careTaker: Boolean,
    val date: LocalDate?,
    val location: Location,
    val qrCode_id: String,                    // QR verification code
    val status: String?,                      // Status: waiting, inprogress
    val typeofAppointment: String
)
```

### Report (from Backend - Port 5005)
```kotlin
data class Report(
    val _id: String,                          // MongoDB ObjectId
    val createdAt: LocalDate                  // Created by doctor/nurse
)
```

## Backend API Integration

This app communicates with the CareHouse Backend through three main API layers:

### Patient Gateway (Port 5000)
- `POST /auth/register` - Patient registration
- `POST /auth/login` - Patient sign-in with JWT
- `POST /auth/logout` - Clear session
- `GET /token` - Refresh access token
- `PUT /patients/:id` - Update patient profile

### Microservices (Internal routing through Gateway)
- **Patient Service (5003)** - User profiles and authentication
- **Appointment Service (5001)** - Appointment CRUD and status tracking
- **Medical Folder Service (5002)** - Access to medical records
- **File Service (5004)** - Document and image uploads/downloads
- **Report Service (5005)** - Medical report retrieval

### Authentication
- **JWT Tokens**: Access tokens (10 min) + Refresh tokens (24 hrs)
- **Token Storage**: Secure cookie persistence
- **Security**: bcrypt password hashing (backend)
- **Authorization**: Role-based (Patient role ID: 2002)

## Getting Started

### Prerequisites
- Android Studio (latest version)
- JDK 1.8 or higher
- Android SDK 26 or higher
- **CareHouse Backend** running and configured (see [Backend Setup](https://github.com/Mohamed-Nafae/Backend-Healthcare-Management-System-for-CareHouse-Project/README.md))

### Backend Setup (Required)
Before running this app, you must set up the CareHouse Backend:

1. **Start MongoDB:**
   ```bash
   mongod
   ```

2. **Start Backend Services:**
   ```bash
   # Terminal 1: Patient Gateway
   cd Patient/patient_interface_BackEnd && npm start
   
   # Terminal 2: Other microservices
   cd Patient/patient_service && npm start
   cd Patient/make_appoinment_service && npm start
   cd Patient/medical_folder_service && npm start
   cd Patient/report_service && npm start
   cd Patient/uplaod_file_service && npm start
   ```

3. **Verify Backend is Running:**
   ```bash
   curl http://localhost:5000/
   ```

### Mobile App Setup

1. Clone the repository
2. Open the project in Android Studio
3. Sync Gradle files
4. Configure backend URL in `data/remote/` API classes (if not localhost)
5. Run the app on an emulator or physical device

### Configuration

Update the base URL in your API configuration if backend is not on `http://localhost:5000`:

```kotlin
// In your Retrofit configuration
const val BASE_URL = "http://localhost:5000/"  // Change for production
```

### Building

**Debug build:**
```bash
./gradlew assembleDebug
```

**Release build:**
```bash
./gradlew assembleRelease
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Connection refused errors | Ensure backend is running on `localhost:5000` |
| JWT token expires | App automatically refreshes token using refresh token endpoint |
| File upload fails | Check backend File Service (Port 5004) is running |
| Appointments not loading | Verify Appointment Service (Port 5001) connectivity |
| Login fails | Check Patient Service (Port 5003) and MongoDB connection |
| Profile pictures not loading | Ensure File Service (Port 5004) GridFS is properly configured |
